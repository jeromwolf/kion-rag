## 3. 벡터 DB 선택 (의사결정 가이드)

```
Q1. 장비 데이터가 10만건 이상인가요?
    → Yes: Milvus 고려
    → No: Q2로

Q2. 기존에 PostgreSQL을 사용 중인가요?
    → Yes: pgvector 고려
    → No: Q3로

Q3. 키워드+벡터 하이브리드 검색이 중요한가요?
    → Yes: Weaviate 고려
    → No: Q4로

Q4. 메타데이터 필터링(웨이퍼, 온도, 기관)이 핵심인가요?
    → Yes: Qdrant 고려
```

### 벡터DB 후보 요약
| 벡터DB | 데이터 규모 | 특징 | 적합 케이스 |
|--------|-------------|------|-------------|
| Qdrant | 1K~100M | 메타데이터 필터링 강력 | KION 기본 권장 후보 |
| Milvus | 100K~1B+ | 대규모, GPU 가속 | 10만건 이상 |
| Weaviate | 1K~100M | 하이브리드 검색 | 장비명+스펙 동시 검색 |
| pgvector | ~100K | PostgreSQL 확장 | 기존 PG 인프라 활용 |
